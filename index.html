¿Que es Docker?

Docker es un sistema operativo (o runtime) para contenedores.
El motor de Docker se instala en cada servidor en el que desee ejecutar contenedores y proporciona un conjunto sencillo de comandos que puede utilizar para crear, iniciar o detener contenedores.

¿Para que se utiliza Docker?

Docker es una plataforma de contenedores que permite a los desarrolladores empaquetar y distribuir aplicaciones de manera eficiente y segura.
Los contenedores aislan los componentes de la aplicación y garantizan que funcionen de manera consistente en cualquier entorno.
Docker también facilita la automatización del despliegue de aplicaciones y la colaboración entre equipos de desarrollo a través de imágenes de contenedor compartidas.

¿Contenedores e Imagenes?

Un contenedor es una instancia de una imagen de Docker que se está ejecutando. Una imagen de Docker es un paquete que contiene todo lo que una aplicación necesita para funcionar, incluyendo el código fuente, las bibliotecas, los archivos de configuración, y más.
Las imágenes se crean a partir de un archivo de configuración llamado Dockerfile, que específica los componentes que deben incluirse en la imagen. Una vez creada una imagen, puede ser descargada y ejecutada en cualquier máquina que tenga Docker instalado. Al ejecutar una imagen, se crea un contenedor que es una instancia única de esa imagen.
En resumen, una imagen de Docker es un paquete estático que contiene todo lo que una aplicación necesita para funcionar, mientras que un contenedor es una instancia en ejecución de esa imagen
